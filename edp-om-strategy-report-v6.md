## 一、方法論定義

### 核心精神

> **不是盲目預測未來，而是先定義「成功的形狀」，再用最小成本逐步驗證猜測是否成立。**

採用 Rita McGrath & Ian MacMillan 的「探索驅動計畫（EDP）」作為底層決策架構。

### ⚠️ 本版重大發現（先讀重點）

```
v5版假設：淨利率30% → 2.4億利潤目標 對應 8億收入規模（已驗證合理）

v6版修正：淨利率15%、毛利率25%（依實際成本結構拆解）
         → 同樣8億收入規模下，淨利只剩約1.2億
         → 與2.4億目標產生 1.2億 缺口

這不是失敗，這是EDP的核心價值：
  「在花錢之前，先發現假設衝突」

本版將此缺口列為最優先驗證假設（見第六章、第八章F-1）

### 三大操作工具

| 工具 | 功能 | 本計畫章節 |
|------|------|----------|
| **逆向財務報表** | 從目標利潤倒推所需規模 | 第六章 |
| **關鍵假設清單** | 記錄「若這個假設錯了整個計畫就垮了」的猜測 | 第八章 |
| **里程碑驗證門** | 不驗證假設，不跨越里程碑，不撥下一筆預算 | 第九章 |

### 戰略定位

| 維度 | 定義 |
|------|------|
| **核心定位** | 成本領導型O&M（全聯模型）：高效率 + 可擴展流程 + 機構級服務品質 |
| **主力客戶** | 策略型機構投資人、大型場站業主（>1MW） |
| **核心差異化** | 隱性損失可視化、PR保證可稽核、數位化異常追蹤 |
| **三層技術架構** | Tier 1：現場工程師 ／ Tier 2：雲端管理平台 ／ Tier 3：進能服平台 |

## 三、成本與毛利結構（新）

> 本章為v6新增核心內容：將公司整體毛利率（25%）拆解為三條內部營運活動線，
> 每條活動線有各自的毛利率，加權組合後形成公司整體毛利率。

### 3.1 毛利率組成（三大業務線）

| 業務線 | 定義 | 營收佔比 | 各業務線毛利率 | 對整體毛利率貢獻 |
|--------|------|---------|-------------|---------------|
| **維運服務**<br>(O&M Service) | 監控、巡檢排班、客戶關係維護、報告產出等常態性服務 | **50%** | **30%** | 15.0% |
| **工程修繕**<br>(Engineering Repair) | 故障排除、零件更換、緊急維修 | **25%** | **25%** | 6.25% |
| **維運工程**<br>(Maintenance Engineering) | 預防性維護工程、清洗作業、設備升級工程 | **25%** | **15%** | 3.75% |
| **加總** | — | **100%** | **加權25%** ✓ | **25%** |

### 3.2 完整利潤結構瀑布圖

```
營收（Revenue）                          100%
  │
  ├─ 營業成本（COGS）                    -75%
  │    ├─ 維運服務成本（佔營收50%中的70%）
  │    ├─ 工程修繕成本（佔營收25%中的75%）
  │    └─ 維運工程成本（佔營收25%中的85%）
  │
  ▼
毛利（Gross Profit）                      25%  ← 三業務線加權平均
  │
  ├─ 營業費用（SG&A：管理、行銷、平台、間接人力） -10%
  │
  ▼
淨利（Net Profit）                        15%  ← 修正後新基準
```

### 3.3 三業務線 × M/OM/AM 服務層級的關聯

```
這兩個分類維度是「正交」的（互相獨立）：

  M/OM/AM   = 對外的「服務承諾深度」分類（客戶看到的）
  維運服務／工程修繕／維運工程 = 對內的「成本活動」分類（管理用的）

舉例：
  一個AM級合約，內部執行時仍會拆解為：
    · 維運服務部分（月報、監控、客戶溝通）── 30%毛利
    · 工程修繕部分（PR未達標時的補修）──── 25%毛利
    · 維運工程部分（PT200清洗排程）──────── 15%毛利

  → AM合約的整體毛利率，取決於三者在該合約中的實際工時與成本配比
  → 長期優化方向：提高「維運服務」佔比（毛利最高），
    降低「維運工程」佔比（可用PT200自動化降低人工投入，提升此線毛利率）
```

### 3.4 毛利結構優化路徑（中長期）

| 業務線 | 現況毛利率 | 優化槓桿 | 目標毛利率（長期） |
|--------|----------|---------|-----------------|
| 維運服務 | 30% | 進能服平台自動化報告，降低人力工時 | 35% |
| 工程修繕 | 25% | 標準化SOP、零件集中採購 | 28% |
| 維運工程 | 15% | **PT200清掃機器人**降低人工清洗成本 | 22% |
| **整體毛利率** | **25%** | 三線同步優化 | **約28~30%（長期願景）** |

> 若整體毛利率能透過上述優化路徑提升至28~30%，淨利率有機會回升至18~20%，
> 將大幅縮小第六章的利潤缺口。**此為B-7假設（見第八章）**。

## 四、PR目標框架（依服務層級）

> 本版修正：依M/OM/AM三層服務，設定不同的PR（Performance Ratio）保證等級。  
> PR值越高，代表場站實際發電量越接近理論最大值，是O&M服務品質的核心指標。

### 4.1 三層PR目標定義

| 服務代號 | 合約類型 | PR目標值 | 承諾性質 | 說明 |
|---------|---------|---------|---------|------|
| **M** | 一般合約 | **PR ≥ 80** | 參考值（不保證） | 基礎維護下的合理期望值，作為服務基準線 |
| **OM** | 保證PR值合約 | **PR ≥ 82** | **保證**（未達標需改善計畫） | 透過妥善率管理與預防性維護達成 |
| **AM** | 保證發電量合約 | **PR ≥ 84** | **保證 + 賠償條款** | 透過進能服平台可稽核報告 + 發電量差額補償機制 |

### 4.2 PR目標與費率的對應邏輯

```
PR 80  ──→  M層級（180,000元/MW）  ──→  一般合約，無保證責任
PR 82  ──→  OM層級（360,000元/MW） ──→  保證PR值，未達標需提出改善計畫
PR 84  ──→  AM層級（600,000元/MW） ──→  保證發電量，未達標啟動賠償

PR每提升2個百分點，費率提升約2倍 (180k→360k→600k)
= PR保證等級是費率差異的核心定價依據
```

### 4.3 PR達標路徑：80 → 82 → 84

| PR提升幅度 | 達成手段 | 對應業務線（第三章） |
|-----------|---------|-------------------|
| 80 → 82（+2） | 強化巡檢頻率、異常快速排除 | 工程修繕 + 維運服務 |
| 82 → 84（+2） | PT200自動清掃 + 進能服平台預測性維護 | 維運工程 + Tier 3技術支援 |

### 4.4 PR保證的風險對價

| 服務層級 | PR保證 | 若未達標 | 風險對價（費率溢價邏輯） |
|---------|--------|---------|---------------------|
| M | PR 80（參考） | 無責任 | 費率最低（180,000元/MW）反映無風險承擔 |
| OM | PR 82（保證） | 提出改善計畫，限期改善 | 費率中等（360,000元/MW）反映管理責任 |
| AM | PR 84（保證+賠償） | 發電量差額補償 | 費率最高（600,000元/MW）反映財務風險承擔 |

## 六、逆向財務報表｜Margin重算＋假設衝突分析

> EDP核心原則：先定義「要賺多少」，再往回推「需要多少規模」。
> **本章是v6最關鍵的修正：套用新margin結構後，發現原利潤目標與原收入規模產生衝突。**

### 6.1 新Margin結構回顧

| 指標 | 數值 |
|------|------|
| **毛利率** | 25%（三業務線加權：維運服務30%×50% + 工程修繕25%×25% + 維運工程15%×25%） |
| **淨利率** | 15% |
| **營業費用率（SG&A）** | 10%（= 毛利率 - 淨利率） |

---

### 6.2 ⚠️ 假設衝突發現：兩個互斥的逆向推論

#### 情境A｜維持原利潤目標（2.4億／四年累計）

```
逆向推論：
  要求的利潤 = 2.4億
  淨利率     = 15%
  ──────────────────
  需要的收入 = 2.4億 ÷ 15% = 16.0億   ← 是v5版8億的「兩倍」

對應容量需求：
  2029總容量需求 ≈ 640MW × 2 ≈ 1,280 MW
  對應人力需求 ≈ 1,280MW ÷ 25MW/人 ≈ 51人  ← 四年內從15人成長至51人

【EDP判定】：此路徑在4年期、15人起點的前提下，規模成長速度
            （4年成長3.4倍人力）已超出組織可承受範圍，
            視為「不可行路徑」，但可作為「5~8年期長期願景」保留。
```

#### 情境B｜維持原收入規模（8億／四年累計，沿用v5容量模型）

```
逆向推論：
  收入規模（沿用v5 MW模型）= 7.98億 ≈ 8億（不變）
  毛利率 = 25% → 四年累計毛利 = 2.0億
  淨利率 = 15% → 四年累計淨利 = 1.2億   ← 這是新的「實際可達」利潤

對應投資回報：
  容許的總投資（沿用v5）= 5.6億
  ROA = 1.2億 ÷ 5.6億 = 21.4%   （v5原為43%）

【EDP判定】：此路徑與v5容量／人力模型（330→610→690→640MW，
            15→25→28→26人）完全相容，不需重新規劃容量與人力。
            但需要正式調整「利潤目標」：從2.4億 → 1.2億。
```

---

### 6.3 建議路徑：採用情境B為基準，情境A列為長期願景

| | 情境B（4年基準，採用） | 情境A（長期願景，5~8年） |
|--|---------------------|----------------------|
| 累計利潤 | **1.2億** | 2.4億 |
| 累計收入 | 8.0億 | 16.0億 |
| 2029年容量 | 640 MW | ~1,280 MW |
| 2029年人力 | 26人 | ~51人 |
| ROA | 21.4% | 43% |
| 可行性 | ✅ 與現有容量/人力規劃相容 | ⚠️ 需5~8年分階段達成 |

> **建議行動**：將「2.4億利潤」重新定位為**8年期目標**（2026~2033），
> 四年期（2026~2029）的中期檢查點為**1.2億累計淨利**。
> 第二個四年期（2030~2033）需再複製一次類似的容量倍增曲線（640→1,280MW）。
